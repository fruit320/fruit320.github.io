有源码的fuzzing（afl-training)

环境构建

apt install clang gcc llvm afl

afl-fuzz使用 常规格式：

afl-fuzz -i input -o out /path/to/program [params]

harness

该练习中有两个测试函数：lib_echo和lib_mul

测试lib_echo 编写harness

AFL_HARDEN=1 afl-clang-fast harness.c library.c -o harness

afl-fuzz -i in -o out ./harness

很快会发现crashes

为了测试lib_mul 编写hardness

CROMU_00007

编译：CC=afl-clang-fast AFL_HARDEN=1 make

将输入放入input文件夹中 （不用编写harness)

afl-fuzz -i in -o out ./cromu_00007

尝试放一个随机文件 aaaa

由于AFL具有启发式算法 所以能帮我们确定这个输入格式（当然会引入时间代价）
